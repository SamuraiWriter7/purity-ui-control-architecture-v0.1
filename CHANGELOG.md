### Core Concepts

* Defines Purity UI as a creator-controlled origin interface, not merely a score display.
* Introduces the initial control architecture layers:

  * Epicenter Layer
  * Proto-Friction Layer
  * Visibility Protocol
  * Circle Versioning
  * No-Inference Layer
  * Royalty OS Visibility
  * Epicenter Network
  * Creator Control Dashboard
* Establishes the distinction between:

  * Purity Detection as a scoring, warning, and review-routing layer
  * Purity UI Control as a visibility, inference, disclosure, and creator-sovereignty layer

### Design Principles

* Creator-defined boundaries should remain authoritative.
* AI systems may assist, read, retrieve, or summarize only within permitted boundaries.
* Unresolved or pre-verbal friction should not be forcibly completed by AI.
* Private creator context should remain private by default.
* Royalty-related UI should show allocation-readiness or estimation only, not guaranteed payment.
* Epicenter network visualization should map meaning relationships, not rank creators.
